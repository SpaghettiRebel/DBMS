ИНСТРУКЦИЯ ПО ЗАПУСКУ И ПРОВЕРКЕ СУБД  (В СРЕДЕ CLION И КОНСОЛИ)
================================================================================

Как запустить и проверить в CLion:
1. Откройте CLion -> Выберите конфигурацию запуска `dbms_http_server`.
2. Перейдите в Edit Configurations... (Изменить конфигурации).
3. В поле "Program arguments" вставьте:
   --data-root ./test_data --host 127.0.0.1 --port 18080
4. Запустите сервер (кнопка Run / зелёная стрелочка).
5. Перейдите к конфигурации `dbms_client` -> Edit Configurations...
   * Убедитесь, что поле "Environment variables" пустое.
6. Запустите `dbms_client`. В консоли CLion появится приглашение `dbms>`.
7. Скопируйте и выполните следующие команды:

    // создает базу данных standard_demo.
    CREATE DATABASE standard_demo;
    // выбирает базу данных standard_demo для дальнейших команд.
    USE standard_demo;
    // создает таблицу users с индексом по id.
    CREATE TABLE users (id INT INDEXED, name STRING, age INT, city STRING);
    // добавляет три записи в таблицу users.
    INSERT INTO users (id, name, age, city) VALUES (1, "Ann", 20, "Moscow"), (2, "Bob", 30, "Kazan"), (3, "Kate", 22, "Moscow");
    // выводит все строки и столбцы таблицы users.
    SELECT * FROM users;
    // выводит id и имя пользователя с id 2 с псевдонимами столбцов.
    SELECT id AS user_id, name AS user_name FROM users WHERE id == 2;
    // выводит имена и возраст пользователей с id от 2 с сортировкой по id.
    SELECT name AS user_name, age AS user_age FROM users WHERE id >= 2 ORDER BY id;
    // обновляет возраст и имя пользователя с id 2.
    UPDATE users SET age = 31, name = "Robert" WHERE id == 2;
    // проверяет обновленные данные пользователя с id 2.
    SELECT id, name, age FROM users WHERE id == 2;
    // считает общее количество строк в таблице users.
    SELECT COUNT(*) AS total FROM users;
    // считает сумму возрастов всех пользователей.
    SELECT SUM(age) AS age_sum FROM users;
    // считает средний возраст пользователя с id 1.
    SELECT AVG(age) AS avg_age FROM users WHERE id == 1;
    // группирует пользователей по городам и считает их количество.
    SELECT city, COUNT(*) AS users_count FROM users GROUP BY city ORDER BY city;
    // ищет id пользователей от 20 лет из городов, начинающихся на mos.
    SELECT id AS matched_id FROM users WHERE age >= 20 AND city LIKE "Mos%";
    // удаляет пользователя с id 1.
    DELETE FROM users WHERE id == 1;
    // выводит оставшихся пользователей с сортировкой по id.
    SELECT * FROM users ORDER BY id;
    // откатывает таблицу users к указанному моменту времени.
    REVERT users "2000.01.01-00:00:00.000";
    // выводит таблицу users после отката.
    SELECT * FROM users;
    // завершает работу клиента.
    exit

8. Проверьте результат: строка с id=2 получит значения по умолчанию ("Guest" и 18).
9. Проверьте файл `dbms_project/server_access.log` — там появятся записи со статусом OK(200).

1. Получить JWT Токен

В PowerShell:

$login = Invoke-RestMethod -Uri "http://127.0.0.1:18080/login" -Method Post -ContentType "application/json" -Body '{"login":"admin","password":"admin"}'
$token = $login.token
$headers = @{Authorization="Bearer $token"}
Проверка:

$login
Должно быть видно:

is_admin : True
token    : eyJ...
Что сказать:

Сначала администратор проходит аутентификацию через /login и получает JWT-токен. Дальше токен передается в Authorization header.
2. Подготовить Базу Для Демонстрации

Лучше каждый раз брать новое имя БД, например:

$db = "advdemo_final"
Дальше:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"CREATE DATABASE $db;`",`"database`":`"$db`"}"

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"USE $db;`",`"database`":`"$db`"}"

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"CREATE TABLE users (id INT INDEXED, name STRING, age INT);`",`"database`":`"$db`"}"

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"INSERT INTO users (id, name, age) VALUES (1, \`"Ann\`", 20), (2, \`"Bob\`", 30);`",`"database`":`"$db`"}"

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"SELECT * FROM users;`",`"database`":`"$db`"}"
Должно быть:

Ann 20
Bob 30
Что сказать:

Я создаю тестовую базу, таблицу и данные через тот же HTTP API, уже с JWT-авторизацией.
3. Пункт 4: Кластер / Шардирование

Invoke-RestMethod -Uri "http://127.0.0.1:18080/cluster/nodes" -Headers $headers | Format-List
Должно быть примерно:

node_id           : shard_0
status            : ONLINE
requests          : ...
errors            : 0
last_heartbeat_ms : ...

node_id           : shard_1
status            : ONLINE
requests          : ...
errors            : 0
last_heartbeat_ms : ...
Что сказать:

Сервер запущен в режиме --cluster-shards 2. Видно два shard-узла, оба находятся в состоянии ONLINE. Запросы маршрутизируются по shard-узлам, статистика запросов хранится отдельно по каждому узлу.
4. Пункт 5: Heartbeat

Проверяется тем же endpoint:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/cluster/nodes" -Headers $headers | Select-Object node_id,status,last_heartbeat_ms
Должно быть:

node_id  status  last_heartbeat_ms
-------  ------  -----------------
shard_0  ONLINE  ...
shard_1  ONLINE  ...
Что сказать:

Heartbeat отражается через статус узла и поле last_heartbeat_ms. Если узел доступен, он находится в состоянии ONLINE, а время последнего heartbeat обновляется.
5. Пункт 8: Telemetry

Invoke-RestMethod -Uri "http://127.0.0.1:18080/cluster/stats" -Headers $headers | Format-List
Должно быть примерно:

mode                     : sharded
total_nodes              : 2
online_nodes             : 2
total_requests           : ...
total_errors             : 0
current_rps              : ...
avg_rps_10min            : ...
max_rps_10min            : ...
avg_response_time_10s_ms : ...
error_rate_1min_pct      : 0
Что сказать:

Endpoint /cluster/stats показывает телеметрию: режим работы, количество узлов, число запросов, ошибки, RPS и среднее время ответа.
6. Пункт 6: Async Queue

Создать асинхронный запрос:

$async = Invoke-RestMethod -Uri "http://127.0.0.1:18080/query_async" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"query`":`"SELECT COUNT(*) AS total FROM users;`",`"database`":`"$db`"}"
$async
Должно быть:

request_id : ...
status     : queued
Получить результат:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/async/$($async.request_id)" -Headers $headers | Format-List
Должно быть:

request_id : ...
status     : completed
result     : {@{total=2}}
Что сказать:

/query_async не выполняет запрос синхронно, а кладет его в очередь и возвращает request_id. После этого по /async/<request_id> можно получить статус и результат выполнения.
7. Пункт 9: JWT / RBAC

Создать пользователя только для чтения:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/accounts/create" -Method Post -ContentType "application/json" -Headers $headers -Body '{"username":"reader","password":"reader","is_admin":false}'
Выдать ему только read-права на БД:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/permissions/set" -Method Post -ContentType "application/json" -Headers $headers -Body "{`"database`":`"$db`",`"target_type`":`"user`",`"target_name`":`"reader`",`"permissions`":{`"read`":true,`"write`":false,`"create_table`":false,`"drop_table`":false,`"drop_database`":false}}"
Залогиниться под reader:

$readerLogin = Invoke-RestMethod -Uri "http://127.0.0.1:18080/login" -Method Post -ContentType "application/json" -Body '{"login":"reader","password":"reader"}'
$readerHeaders = @{Authorization="Bearer $($readerLogin.token)"}
Проверить, что SELECT разрешен:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $readerHeaders -Body "{`"query`":`"SELECT * FROM users;`",`"database`":`"$db`"}"
Проверить, что UPDATE запрещен:

Invoke-RestMethod -Uri "http://127.0.0.1:18080/query" -Method Post -ContentType "application/json" -Headers $readerHeaders -Body "{`"query`":`"UPDATE users SET age = 99 WHERE id == 1;`",`"database`":`"$db`"}"
Ожидаемо:

403 forbidden_by_rbac
